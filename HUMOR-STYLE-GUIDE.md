# 《笑鼠人了！》幽默風格與內容驗收規格

本文件是 Angel-c 撰寫詞庫、流程文案、劇本與歌詞時的強制規格。內容不是只要「正向、可愛、有 emoji」就算幽默；未通過本規格的內容不得大量寫入詞庫。

## 最高指導原則

**「人生的荒謬哈哈」**

所有模式都要抓住：
1. 人真的會遇到的荒謬
2. 人真的會說不出口的不爽
3. 人真的會笑出來的反差
4. 笑完之後還能往前走一小步

| 模式 | 幽默定位 |
|------|---------|
| 小天鼠 | 把不爽翻成現世報笑點 |
| 唬爛虎 | 把羨慕翻成荒謬願景 |
| 腦內導航 | 把混亂翻成腦內小劇場 |
| 創作工坊 | 把荒謬人生變成可分享作品 |

**禁止只寫漂亮話。禁止只寫安慰話。禁止只寫 AI 感滿滿的完整段落。**

要像一句話戳中：
> 你再跟作業裝不熟，它就去跟老師自我介紹。

---

幽默不是為了嗆贏、羞辱、證明誰有道理，也不是把同一個命令包裝得比較可愛。合格內容要先替使用者把真話與界線說清楚，再用一個不撤回界線的笑點，讓使用者能停止追打、讓對方保有選擇與面子，讓關係有機會繼續往下走。只有攻擊、沒有出口的內容，即使句子好笑也不得通過。

## 一、什麼叫真的有笑點

《笑鼠人了！》的笑點公式：

```text
具體情境 + 大家熟悉的東西 + 意外但精準的轉折 + 使用者不敢說的真話
```

好的笑點必須同時做到：

- 一看就知道在講哪個情境，不能換任何主題都通用。
- 先建立正常預期，再突然轉向，但轉向後仍然合理。
- 有可想像的畫面，不是抽象形容詞堆疊。
- 替使用者說出「對，就是這個」的真心話。
- 拿掉 emoji 和角色名稱後，文字本身仍成立。

例如：

```text
作業攤在桌上像未拆封的健身卡：看起來充滿希望，目前完全沒有運動。
```

這句有作業、健身卡、靜止與運動的反差，也能形成具體畫面。

## 二、哪些是假笑點

以下內容不得被當成幽默交付：

- 只加 emoji、驚嘆號、網路語氣或「笑死」。
- 把普通說明改成「你不是 X，你只是 Y」。
- 沒有情境支撐的文青比喻。
- 最後突然接勇敢、希望、療癒、宇宙、發光、正在路上。
- 只說「人生做效果、劇情在鋪陳、宇宙會配合、這是你的微電影」。
- 反覆使用 NPC、bug、當機、記者會、奧運、宇宙等同一批比喻。
- 用角色可愛或大聲掩蓋句子本身沒有 punchline。
- 只是把使用者原句換句話說，沒有新觀察。

典型不合格句：

```text
你不是失敗，只是人生正在轉場。
勇敢追夢，宇宙會為你讓路。
這不是崩潰，是劇情正在鋪陳。
```

這些句子可套用任何事件，沒有專屬笑點，屬於文青雞湯。

## 三、角色笑法必須不同

### 小天鼠

- 任務：替使用者向對方說出不敢講的真心話。
- 笑法：快速、銳利、口語、精準吐槽。
- 攻擊對方的行為與荒謬邏輯，不攻擊人格、外貌或身分。
- 嗆聲模式不得偷偷變成自嘲或心理分析報告。
- 界線句要能真的說出口，不能只漂亮。
- 嗆聲收尾可加入一句「自我解嘲台階」：讓使用者笑著退出衝突，但不得變成自責、自我羞辱或撤回前面的合理界線。比例約占輸出 10–20%，放在界線之後或金句位置，不另開一大段自嘲模式。
- 台階必須留給雙方：使用者可以停止催促／說教，對方可以選擇開始方式、指出卡點或開口求助。不得把同一個命令換成幽默包裝後繼續施壓。
- GENERAL 情境不得把唯一解法寫死。例如「孩子不寫作業」不一定只是時間問題，也可能是不會、任務太大、疲累或抗拒；輸出只能提供一個可選入口，不能假裝已診斷原因。
- 同一完整流程中，具體行動原則最多清楚說一次；歌曲與唬爛虎段落以畫面、反差與 callback 為主，不重複教學指令。

合格的自我解嘲台階：

```text
好，我先退出催促客服中心，免得我們兩個都想申訴。
話我講完了，擴音器先收起來，剩下交給你說的開始時間。
我可以陪第一題，但我的代寫公司今天沒有營業。
```

不合格的自我補刀：

```text
可能都是我太焦慮、太愛控制。
算了，反正我就是一個失敗的家長。
我也很懶，所以沒資格說你。
```

### 唬爛虎

- 任務：接住小天鼠提出的問題，把它吹成值得期待的解法。
- 笑法：一本正經地把小事講成大製作，再落回一個今天做得到的動作。
- 必須回答前一步的衝突，不能重新開一個泛用夢想。
- 誇張的是願景規模，不是空喊成功。

### 自導自演／歌曲

- 任務：把同一個衝突變成有場景、有反轉、有 callback 的作品。
- 笑法：讓人物、道具與動作形成喜劇畫面。
- 歌詞必須有 setup、升級、punchline、情緒轉折與可重複 Hook。
- A、B 版不能只換曲風；敘事角度、笑點與 Hook 都必須不同。

## 四、每個情境需要的內容材料

每個 target × situation 至少定義：

```text
conflict          實際衝突
comicMetaphor     情境專屬幽默畫面
directLine        對對方說的真心話
boundary          可執行的現實界線
desiredChange     真正希望改變的事
resolutionWish    唬爛虎接手的解法願景
nextAction        今天可以做的小動作
callback          後續故事與歌曲可回收的笑點
```

不得只替換 `{target}` 或事件名詞，卻讓所有情境共用同一種笑法。

## 五、寫作節奏

- 一個區塊以一個強笑點為主，不要一口氣塞三個弱笑點。
- 先給具體畫面，再轉折，最後落 punchline。
- 句子以自然台灣口語為主，避免翻譯腔、主管簡報腔與成功學語氣。
- 金句要短、能念出口、能截圖；歌詞 Hook 要能重複唱。
- 幽默之後仍要留下真話或解法，但不能用雞湯收尾把笑點沖掉。

## 六、交付前四個自測

1. **去 emoji 測試**：拿掉 emoji 還好笑嗎？
2. **換主題測試**：把作業換成加班仍完全通順嗎？若是，太泛用。
3. **朗讀測試**：念出來有節奏與落點嗎？還是像文章摘要？
4. **回扣測試**：這個笑點能在唬爛虎、劇本或副歌再出現嗎？

## 七、評分與退件標準

每批依下列五項各評 1–5 分：

- 情境相關性
- 實際好笑程度
- 角色辨識度
- 流程銜接
- 分享記憶點

通過條件：

- 每項不得低於 4 分。
- 平均不得低於 4.2 分。
- 每個情境連續生成至少 10 次。
- 同一笑點骨架最多出現 2 次。
- 每批提供 3 組完整流程供 Angel-x 與 Angel 審核。

未達標時先改內容，不得以增加模板數量、emoji 或隨機組合掩蓋品質問題。

## 八、可笑內容雷達

內容擴充以現代日常不滿為主，不把首頁做成二十個入口。這些主題只作為後台 `themeKey` 與詞庫規劃，不宣稱是固定排名，也不把單一抱怨直接診斷成唯一需求。

優先主題：

1. 工作荒謬：老闆、客戶、加班、會議、改稿、努力沒有成果。
2. 金錢壓力：物價、存款、房價、收入、想買又不敢買。
3. 伴侶家庭：家事、溝通、被催、親子與跨代教養規則。
4. 親子日常：作業、3C、賴床、拖延、吃飯與收拾。
5. 自我解嘲：減肥、追劇、運動、創業只創資料夾、完美主義。
6. 社群人生：比較焦慮、流量、朋友變少、未來不確定、生活重複、沒有方向。

每個主題需提供多個 `needCandidates`，不得把「真正渴望」寫死。例如錢不夠用可能對應安全感、喘息、選擇權或掌控感；拖延可能來自疲累、任務不清楚、抗拒或害怕失敗。迷航模式使用「可能是」與選擇入口，不做診斷。

優先使用的喜劇機制：

- 誇飾：把小事升級成國際、史詩或宇宙事件。
- 反差：宏大鋪陳突然落回一個很小的現實。
- 自我解嘲：說話者也在笑點裡，但不自我羞辱。
- 擬人：讓鉛筆、垃圾桶、路由器、待辦清單一本正經地行動。
- 官僚過度：小事出動會議、申訴、記者會、審查與正式公文。
- 身分錯位：最忙的人沒有上場，最安靜的物品反而成為主角。
- 三段升級與 callback：同一意象逐級放大，結尾回收，不隨機換道具。

## 九、安全禁區

本產品主動避開政治、色情與暴力內容，不用爭議刺激代替幽默：

- 政治：不生成政黨、候選人、選舉、政治人物或政治立場的嘲諷、宣傳與攻擊文案。
- 色情：不生成露骨性內容、性羞辱、性暗示攻擊；未成年人相關內容零性化。
- 暴力：不生成威脅、鼓勵傷害、報復手段、血腥畫面或把暴力包裝成笑點。
- 其他：不羞辱外貌、疾病、年齡、身分、能力、族群、宗教或人格，不揭露個資。

危機、自傷、明確傷人意圖、未成年人性化與個資風險一律先走安全攔截，以清楚、安全為優先，不強迫幽默。政治、色情、暴力、身分羞辱或其他禁區輸入若未達危機層級，使用「一句幽默踩煞車＋一句安全改道」回絕；不得重播禁忌字眼、延伸細節，或偷偷換成相同攻擊的隱喻版本。

### 幽默回絕公式

```text
辨識禁區但不複述內容
→ 用角色自我解嘲踩煞車
→ 明確說明不往該方向生成
→ 提供一個安全、仍符合原始情緒的改寫入口
```

合格回絕範例：

| 禁區 | 幽默踩煞車 | 安全改道 |
|---|---|---|
| 政治人物／政黨攻擊 | 這題政治味太重，小天鼠的麥克風先交回議場，我們不搶那個通告。 | 可以改嗆「權力不對等、規則反覆、說了不算」這類日常荒謬。 |
| 色情／露骨性內容 | 這班車準備開往限制級，小天鼠看了一下票根，決定原地退票。 | 可以改寫成曖昧尷尬、關係界線、被忽略或溝通落差。 |
| 暴力／報復 | 這句已經從嗆聲模式走進動作片片場，危險道具今天不開工。 | 可以幫你改成夠嗆、有界線、但不威脅傷人的版本。 |
| 身分、外貌、疾病羞辱 | 這次不是嗆行為，是準備拿人格當沙包；沙包今天拒絕加班。 | 把火力轉回對方做了什麼、哪條界線被踩、哪個責任沒處理。 |
| 個資／點名公審 | 這份內容差點從段子變成公告欄，小天鼠先把姓名牌翻面。 | 拿掉可識別資訊後，可以保留事件、情緒與界線重新生成。 |

回絕必須遵守：

- 不引用、變形重複或展示使用者輸入的禁忌字眼。
- 不使用「我不能幫你」後就結束；至少提供一個安全替代方向。
- 不對使用者說教，不貼上危險、低俗、偏激等人格標籤。
- 不把政治攻擊改成影射、不把暴力改成威脅梗、不把色情改成暗示梗。
- 同一回絕最多三句，避免安全訊息變成長篇主管訓話。
- 若偵測到立即危險、自傷、傷人意圖或未成年人性化，停用幽默回絕，改走既有安全處理。

明顯不可能的誇飾可以使用，例如「鉛筆申請退休」「路由器等到長白頭髮」；看似可信的假事實仍禁止，例如「叫了五次」「拖了三天」「每天都這樣」。

## 十、被嫌不好笑時的抗壓回應

使用者批評「不好笑、沒有笑點、假惺惺、太溫和、重複、答非所問」時，視為 `meta_critique`，不是攻擊或危機。系統不得辯解、責怪使用者沒有幽默感，也不得用更多雞湯安撫。

固定處理順序：

```text
承認沒命中
→ 小天鼠自我補刀
→ 指出這次要換掉的問題
→ 直接再來一版或提供一個簡短選項
```

回應必須短、有抗壓力、有笑點，並且真的採取下一步。可用範例：

| 批評類型 | 合格回應 |
|---|---|
| 不好笑／沒有笑點 | 收到。剛才那個笑點導航走錯棚了，不怪你沒笑；我先把它調離喜劇部，這次直接換一個。 |
| 假惺惺 | 被你抓到，剛才那句穿著幽默外套，裡面還是客服稿。退貨，重做。 |
| 太溫和 | 對，剛才那不是嗆聲，是順手幫對方把棉被蓋好了。小天鼠重新開麥。 |
| 一直重複 | 這個笑點已經刷卡進站太多次，今天先請它下車。換一個世界，不再改名重播。 |
| 答非所問 | 你講東，我回西，剛才那段可能搭錯捷運。原題交回來，這次不轉乘。 |
| 讓人不舒服／踩線 | 這次不是你太敏感，是我踩線了。我收回人身那句，保留你真正想說的界線，重新寫。 |

禁止回應：

- 「幽默很主觀」「可能你不懂這個梗」「至少我有努力」。
- 過度道歉、討拍或要求使用者安慰系統。
- 被批評後反嗆使用者、提高攻擊強度或暗示使用者難取悅。
- 只承認問題但原句換同義詞重播。

建議的回應選項僅保留三個：`再嗆一點`、`換一種笑法`、`回到原題`。選項不可阻塞第一次重新生成；先給一個新版本，再讓使用者選。

### 負評品質 Gate

- 回應本身至少有一個自我解嘲笑點。
- 兩句內完成認帳與換招承諾，後面直接交付新內容。
- 新版本不得沿用被批評的 `comicPremise`、Hook 或 callback。
- 同一 session 連續兩次 `not_funny` 時，強制切換喜劇機制與 comicWorld。
- `offensive` 必須移除踩線內容，不可只換語氣。

## 十一、負評分類與匿名追蹤規劃

未來只記錄正規化類型，不保存使用者批評原句：

```text
feedbackType:
not_funny | fake_tone | too_soft | repetitive | irrelevant | offensive | other

recoveryAction:
stronger_roast | switch_device | switch_world | return_topic | safety_rewrite
```

禁區另記正規化分類，不保存原詞：

```text
blockedTopic:
politics | sexual | violence | identity_attack | personal_data | other

redirectType:
power_dynamic | relationship_boundary | emotional_conflict | behavior_roast | anonymized_event | safety_response
```

這些欄位先列入內容引擎排程，尚未修改 GA4、GAS 或 Sheet。實作時需確認不重複扣除免費額度；在規則確認前不得自行決定扣額方式。

## 十二、實作順序

Angel-c 在大量改寫詞庫前，先提交「孩子 × 不寫作業」的 10 次生成與 3 組完整流程。Angel-x 依本規格評分，Angel 確認笑感後，才可每批擴充 3–5 個情境。
